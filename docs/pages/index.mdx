# agentful

Claude Code configuration that adds specialized agents, quality gates, and structured workflows to your project.

**[Quick Start Guide →](/getting-started/quick-start)**

---

## What It Does

agentful extends Claude Code with:

1. **Specialized agents** - Backend, frontend, testing, review, orchestration
2. **Quality gates** - Type checking, linting, test coverage, dead code detection
3. **Product tracking** - Feature status, completion percentage, blocking decisions
4. **Smart analysis** - Tech stack detection, pattern recognition, domain discovery

It's a `.claude/` directory structure with agents, commands, and skills that Claude Code loads.

---

## How It Works

### Agent Architecture

Claude Code runs with different instruction sets depending on which agent you invoke:

- `@orchestrator` - Coordinates work, never writes code
- `@architect` - Analyzes tech stack, generates domain agents
- `@backend` - Server-side logic, APIs, databases
- `@frontend` - UI components, state management, client code
- `@tester` - Test generation and execution
- `@reviewer` - Quality validation, security checks
- `@fixer` - Automated remediation of failures

Each agent has explicit scope boundaries and responsibilities.

### Workflow

```
1. Define product spec (.claude/product/)
2. Run /agentful-start
3. Orchestrator identifies next task
4. Domain agents implement
5. Tester writes tests
6. Reviewer validates (types, lint, tests, coverage)
7. Fixer remediates failures
8. Repeat until complete
```

Human checkpoints pause for architectural decisions, library selection, security configurations.

## Installation

**Prerequisites:**
- [Claude Code](https://code.anthropic.com)
- Node.js 22+

**Initialize in your project:**

```bash
npx @itz4blitz/agentful init
```

This creates:
- `.claude/` - Agents, commands, skills (version control this)
- `.agentful/` - Runtime state (gitignored)
- `.claude/product/` - Product specification structure

## Quick Start

### 1. Define Product

**New projects:**

```bash
claude
/agentful-product
```

Answer guided questions. Creates structured `.claude/product/` hierarchy.

**Existing codebases:**

```bash
claude
/agentful-product
```

Reverse-engineers spec from your code.

**Manual:**

Edit `.claude/product/index.md`:

```markdown
## Overview
Task management for distributed teams

## Tech Stack
- Next.js 14 + TypeScript
- Prisma + PostgreSQL
- Vitest + Playwright

## Features
1. Authentication (CRITICAL)
2. Project management (HIGH)
3. Task assignment (MEDIUM)
```

### 2. Generate Agents

```bash
/agentful-generate
```

Analyzes your codebase and generates:
- Domain-specific agents based on your business logic
- Tech stack skills for framework patterns
- Pattern-matched instructions from your actual code

### 3. Start Development

```bash
/agentful-start
```

Orchestrator begins implementation with quality gates.

### 4. Monitor Progress

```bash
/agentful-status
```

Shows completion percentage, current task, quality gate status, pending decisions.

| Command | Purpose |
|---------|---------|
| `/agentful` | Natural language conversation |

| `/agentful-generate` | Analyze codebase, generate agents and skills |
| `/agentful-start` | Begin structured development |
| `/agentful-status` | View progress and quality gates |
| `/agentful-validate` | Run quality checks manually |
| `/agentful-decide` | Answer blocking decisions |

## Quality Gates

All implementations validated against 6 core automated quality gates:

- **Type checking** - TypeScript, Flow, Python types, etc.
- **Linting** - ESLint, Biome, Ruff, etc.
- **Tests** - All suites must pass
- **Coverage** - Minimum 80%
- **Dead code** - Unused exports, files, dependencies
- **Security** - Basic security validation

Gates adapt to your stack. The framework detects available tools and runs appropriate checks.

## New vs Existing Projects

### New Projects (No Code)

1. Architect prompts for tech stack
2. Generates template agents from best practices
3. Implements first feature
4. Re-analyzes actual code
5. Updates agents with your specific patterns
6. Continues with refined agents

### Existing Projects (With Code)

1. Architect samples codebase
2. Detects patterns, conventions, structure
3. Generates agents matching your exact style
4. Begins implementation

## Extended Sessions

Use [Ralph Wiggum plugin](https://github.com/anthropics/claude-code/tree/main/plugins/ralph-wiggum) for uninterrupted development:

```bash
claude
/plugin install ralph-wiggum@anthropics
/ralph-loop "/agentful-start" --max-iterations 50 --completion-promise "AGENTFUL_COMPLETE"
```

Runs until complete, quality gates pass, or human decision required.

---

## What It's Not

- Not fully autonomous - requires human decisions
- Not a testing framework - generates tests, needs infrastructure
- Not a deployment system - development focused
- Not a replacement for developers - accelerates implementation

Works best with clear product specs and timely decisions.
