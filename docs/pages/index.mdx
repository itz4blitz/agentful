## What is agentful?

AI development toolkit that coordinates specialized agents in Claude Code to build software from product specs.

**Key capabilities:**
- **[Parallel execution](/concepts/background-agents)** - Multiple agents work simultaneously in isolated git worktrees (frontend + backend + tests running concurrently)
- **[Three-tier architecture](/agents#three-tier-architecture)** - Core (pre-built), domain (generated for your stack), ephemeral (temporary, task-specific)
- **[Shared skills](/skills/validation)** - Reusable capabilities like validation, testing, and research across all agents
- **Custom hooks** - Intercept and modify agent behavior at key lifecycle points
- **Quality gates** - Automated validation for every change (tests, types, coverage, security)
- **Human checkpoints** - You decide, agents execute

## Our Philosophy

Every software project needs [three tiers of agents and skills](/agents#three-tier-architecture):

1. **[Core](/agents#1-core-agents)** (included) - Universal agents ([orchestrator](/agents/orchestrator), [reviewer](/agents/reviewer), [fixer](/agents/fixer)) + shared [skills](/skills) ([validation](/skills/validation), [testing](/skills/testing), [research](/skills/research)). Same for every project.
2. **[Domain](/agents#2-domain-agents)** (generated) - Stack-specific agents (backend, frontend for web; extractor, transformer for pipelines) + tech skills (Next.js, Postgres, Tailwind). [Architect](/agents/architect) creates these based on your codebase.
3. **[Ephemeral](/agents#3-ephemeral-agents)** (temporary) - Task-specific agents/skills created for one-off tasks or longer-term work, then removed when done (migration-agent for a 2-hour DB migration, performance-profiling for a multi-week optimization project).

**Skills are shared across agents.** Any agent can use validation, research, etc. Our OOTB setup works for all software projects, but use the [custom installer](/getting-started/installation#custom) to tweak what you need.
