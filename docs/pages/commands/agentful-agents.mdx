# /agentful-agents

Discover domain patterns in your codebase and generate specialized domain agents with comprehensive documentation.

## Overview

The `/agentful-agents` command automatically discovers business domains in your codebase and generates specialized agents for them. Unlike tech stack agents (like `nextjs-specialist`) that handle framework patterns, domain agents understand your business logic like billing, inventory, authentication, or content management.

This command:

- Analyzes your code structure to discover logical domains
- Uses parallel exploration agents to identify patterns
- Validates findings with confidence scores
- Generates domain-specific agents with real examples from your code
- Creates comprehensive documentation for each agent
- Updates your architecture metadata

## Usage

### Basic Usage

```bash
/agentful-agents
```

This will:
1. Auto-detect your current project state
2. Run appropriate workflow based on what exists
3. Discover domain patterns (if code exists)
4. Present findings with confidence scores
5. Ask for confirmation before generating anything
6. Generate agents in parallel with validation

### After Architecture Analysis

```bash
# Recommended: Run after initial architecture setup
/agentful-start
# Architecture analyzed, tech stack agents created

/agentful-agents
# Now discover business domain agents
```

### Mode 3: DISCOVERY (Main Workflow)

When codebase exists and architecture is analyzed.

This is the primary workflow with four steps:

#### Step 1: Parallel Domain Exploration

The command spawns four parallel explorer agents:

**1. Cohesion Explorer** - Analyzes file clustering and module coupling
- Looks for file clusters with high internal coupling
- Analyzes import graphs to find interdependent modules
- Identifies folders with cohesive purpose
- Outputs confidence based on coupling strength

**2. Naming Explorer** - Analyzes naming patterns
- Scans for business domain terms in file/folder/class names
- Detects consistent naming patterns
- Finds domain prefixes (BillingService, AuthController)
- Outputs confidence based on naming consistency

**3. Database Explorer** - Analyzes data layer
- Analyzes database schemas and models
- Identifies entity relationships that define domains
- Finds bounded contexts in data layer
- Outputs confidence based on entity cohesion

**4. API Explorer** - Analyzes API structure
- Maps API routes to business capabilities
- Identifies controller/handler groupings
- Finds REST resource patterns
- Outputs confidence based on route organization

Each explorer outputs domain candidates with:
- Domain name
- Confidence score (0.0 - 1.0)
- Evidence supporting the finding
- List of relevant files

#### Step 2: Validate and Aggregate Findings

After explorers complete, a validator agent:
- Merges overlapping domain candidates
- Calculates composite confidence scores
- Filters out low-confidence results (< 60%)
- Checks existing agents in `.claude/agents/`
- Identifies gaps where domains exist but no agent

**Confidence calculation:**
```
composite_confidence = (
  cohesion_score * 0.3 +
  naming_score * 0.2 +
  database_score * 0.3 +
  api_score * 0.2
)

Only include if composite_confidence >= 0.6
```

## Discovery Process

### How Domains Are Discovered

Domains are **never hardcoded**. The command discovers them by analyzing actual code patterns:

**Strong domain signals:**
- Multiple files in dedicated folder (src/billing/)
- Consistent naming across files (BillingService, billing.controller.ts)
- Dedicated API routes (/api/billing/*)
- Related database models (subscriptions, invoices, payments)
- High internal coupling, low external coupling

**Weak domain signals (filtered out):**
- Single file claiming to be a domain
- Generic names (utils, helpers, common)
- Low confidence scores (< 60%)
- No clear boundaries or purpose

### Confidence Scoring

Each domain gets a confidence score:

| Score | Meaning | Action |
|-------|---------|--------|
| 85-100% | Very strong domain | Highly recommend creating agent |
| 70-84% | Strong domain | Recommend creating agent |
| 60-69% | Moderate domain | Consider creating agent |
| < 60% | Weak signal | Filter out, likely false positive |

### What Gets Generated

For each discovered domain:

**1. Domain Agent File** (`.claude/agents/domain.md`)
- Agent definition with frontmatter
- Scope and responsibilities
- Real code examples from your project
- Business rules extracted from code
- Integration patterns
- Error handling strategies
- Database schema documentation

**2. Domain Documentation** (`.claude/agents/domain.README.md`)
- Domain overview
- When to use this agent
- Architecture diagrams
- Common patterns
- Integration points
- Key responsibilities
- Example usage

**3. Architecture Metadata** (`.agentful/architecture.json` updated)
- Domain metadata
- Confidence scores
- File counts
- Key files
- External services
- Database models

## Integration with Other Commands

### Before: /agentful-start

Run `/agentful-start` before `/agentful-agents`:

```bash

/agentful-start

# 2. Implement some features

/agentful-agents
```

**Why this order?**
- Architecture analysis creates `.agentful/architecture.json`
- Implemented features provide code patterns to discover
- Domain agents build on top of tech stack agents

Once agents are generated, use them in development:

```bash
# Mention domain agents in prompts
```

As your codebase grows:

```bash

/agentful-agents # Discover new domains

# Monthly check
/agentful-agents # See if new domains emerged
```

## Tips and Best Practices

### Discovery Tips

1. **Run after implementing features**
   - Wait until you have 20+ files
   - Domain patterns more visible with more code

2. **Re-run periodically**
   - After major features
   - Monthly for active projects
   - When new business capabilities added

3. **Review findings carefully**
   - High confidence (85%+) = strong signal
   - Low confidence (60-69%) = may be false positive
   - Check evidence before confirming

4. **Customize generated agents**
   - They're starting points, not final versions
   - Add project-specific rules
   - Update as business logic evolves

### Agent Usage Tips

1. **Use domain agents for business logic**
   ```bash
   # Good
   "@billing, implement subscription upgrades"
   "@inventory, add low-stock alerts"

   # Not as good
   "@backend, add billing feature"  # Too generic
   ```

2. **Let orchestrator delegate**
   ```bash
   # The orchestrator automatically picks the right agent
   /agentful-start
   # Sees "billing" task â†’ delegates to @billing
   ```

3. **Combine with tech stack agents**
   ```bash
   # Domain agents work with tech stack agents
   "@billing and @nextjs-specialist, create checkout page"
   ```

### Maintenance Tips

1. **Update agents as code evolves**
   - Manually edit `.claude/agents/domain.md`
   - Add new patterns discovered
   - Update business rules

2. **Keep architecture.json current**
   - Re-run `/agentful-agents` periodically
   - Updates metadata automatically

3. **Document integration points**
   - How domains interact
   - Shared data models
   - API contracts

## Troubleshooting

### Issue: "No domains discovered"

**Cause:** Codebase too small or no clear patterns

**Solution:**
```bash
# Check file count
ls -R src/ | wc -l

# If < 20 files, wait until more code exists
/agentful-start  # Implement more features

# Then try again
/agentful-agents
```

### Issue: "Low confidence scores"

**Cause:** Code structure doesn't show clear domain boundaries

**Solution:**
- Organize code into domain folders (src/billing/, src/auth/)
- Use consistent naming patterns
- Group related files together
- Re-run discovery after reorganizing

### Issue: "Generated agent has placeholders"

**Cause:** Validation failed to catch it

**Solution:**
```bash
# Manually fix the agent file
# Or delete and regenerate
rm .claude/agents/domain.md
/agentful-agents  # Regenerate
```

### Issue: "Want to regenerate an agent"

**Cause:** Need to update agent with new patterns

**Solution:**
```bash
# Option 1: Manually edit
# Edit .claude/agents/domain.md

# Option 2: Delete and regenerate
rm .claude/agents/domain.md
rm .claude/agents/domain.README.md
/agentful-agents  # Will detect missing and regenerate
```

### Issue: "Domain exists but not discovered"

**Cause:** Confidence score too low (< 60%)

**Solution:**
- Improve code organization for that domain
- Add more files to the domain
- Use consistent naming
- Or manually create agent in `.claude/agents/`

### Batch Re-Discovery

Check all domains periodically:

```bash
#!/bin/bash
# rediscover-domains.sh

# Compare last run timestamp
LAST_RUN=$(jq -r '.domain_discovery.last_run' .agentful/architecture.json)
DAYS_SINCE=$(( ($(date +%s) - $(date -d "$LAST_RUN" +%s)) / 86400 ))

if [ "$DAYS_SINCE" -gt 30 ]; then
  echo "Running monthly domain discovery..."
  /agentful-agents
else
  echo "Last run $DAYS_SINCE days ago. Skipping."
fi
```
