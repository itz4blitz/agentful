# Installation

## Prerequisites

- Claude Code - [Install here](https://www.claude.com/product/claude-code)
- Node.js 22+
- A project directory

## Installation Options

> **Note**: These options are for **first-time installation only**. If you already have agentful installed, use `/agentful-update` instead. See [When to Use What](#when-to-use-what) below.

### Option 1: Web Configurator (Recommended)

Use the interactive web configurator to customize your installation:

**[agentful.app](https://agentful.app)**

1. Choose a preset or build custom configuration
2. Select agents, skills, hooks, and quality gates
3. Generate installation command
4. Copy and run in your project

See the [Web Configurator Guide](/getting-started/configurator) for detailed instructions.

### Option 2: Default Installation (CLI)

Install with all components (recommended for most projects):

```bash
# Default installation - All components (recommended)
npx @itz4blitz/agentful init

# Minimal installation - Backend-only (2 agents, for simple projects)
npx @itz4blitz/agentful init --preset=minimal

# View installation options
npx @itz4blitz/agentful presets
```

Tech stack is auto-detected on first run - no need to specify during installation.

See the [Installation Options Guide](/getting-started/presets) for details.

### Option 3: Custom Configuration (CLI)

Build a custom configuration with specific components:

```bash
npx @itz4blitz/agentful init \
  --agents=orchestrator,backend,frontend,tester \
  --skills=validation,testing \
  --hooks=health-check \
  --gates=types,tests,coverage
```

**Available flags**:
- `--preset=<name>` - Use a preset configuration
- `--agents=<list>` - Comma-separated agent names
- `--skills=<list>` - Comma-separated skill names
- `--hooks=<list>` - Comma-separated hook identifiers
- `--gates=<list>` - Comma-separated quality gate names

### Option 4: Shareable Configuration

Use a shared configuration from the web configurator:

```bash
npx @itz4blitz/agentful init --config=https://agentful.app/c/abc12345
```

This installs:
- `.claude/` - Agents, commands, skills
- `.agentful/` - Runtime state (gitignored)
- `.claude/product/` - Product specification structure
- `CLAUDE.md` - Project instructions

## Product Structure

agentful uses `.claude/product/index.md` for your product specification.

**Simple projects**:
- Keep all features in `index.md`

**Complex projects**:
- Organize into domain subdirectories under `domains/`
- Each domain has its own `index.md` and `features/`

## Verify Installation

```bash
# Check directories exist
ls -la .claude/
ls -la .agentful/

# Check state
cat .agentful/state.json

# Check product structure
ls -la .claude/product/
cat .claude/product/index.md
```

## Additional Options

### Bare Init

Skip template files if you already have your spec:

```bash
npx @itz4blitz/agentful init --bare --preset=minimal
```

### Skip Smart Analysis

Faster init without project analysis:

```bash
npx @itz4blitz/agentful init --no-smart
```

### Generate Domains

Generate domain subdirectories for complex projects:

```bash
npx @itz4blitz/agentful init --generate-domains
```

### Override Components

Customize the default installation with specific selections:

```bash
# Start with minimal, add frontend
npx @itz4blitz/agentful init \
  --preset=minimal \
  --agents=orchestrator,backend,frontend

# Start with default, customize skills
npx @itz4blitz/agentful init \
  --skills=product-tracking,validation,testing,deployment
```

### Global Install

For frequent use:

```bash
npm install -g @itz4blitz/agentful
agentful init --preset=minimal
```

Note: `npx` always uses the latest version.

## Updating Your Installation

After initial installation, use the `/agentful-update` command to get the latest templates:

```bash
# In Claude Code
/agentful-update
```

This command:
- ğŸ”„ Updates agents, commands, and skills to the latest version
- ğŸ’¾ Preserves your customizations via 3-way merge
- ğŸ“¦ Creates automatic backups before updating
- ğŸ”’ Safely handles configuration conflicts
- âœ¨ Adds new features from agentful updates

### Update Process

1. **Backup**: Automatically backs up current configuration
2. **Fetch**: Downloads latest templates from registry
3. **Merge**: Intelligently merges changes with your customizations
4. **Verify**: Validates merged configuration
5. **Apply**: Updates files if validation passes

### When to Update

Run `/agentful-update` when:
- A new version of agentful is released
- You want new agent capabilities or commands
- Bug fixes are available for existing templates
- You need to sync with upstream improvements

## Troubleshooting

### Command not found: npx

Install Node.js 22+ from [nodejs.org](https://nodejs.org/)

```bash
node --version  # Should be v22+
npx --version
```

### .claude directory already exists

Backup before overwriting:

```bash
mv .claude .claude.backup
npx @itz4blitz/agentful init
```

### Permission denied

```bash
ls -la
sudo chown -R $USER:$USER .claude .agentful
```

### Network timeout

```bash
# Clear cache
npm cache clean --force
npx @itz4blitz/agentful init
```

## Uninstall

```bash
rm -rf .claude .agentful CLAUDE.md
```

## Next Steps

After installation:

1. Edit your product spec: `.claude/product/index.md` ([Quick Start](/getting-started/quick-start))
2. Start Claude Code: `claude`
3. Begin development: `/agentful-start`

To update agentful later, use `/agentful-update` (not `npx init`)
