## Step 1: Initialize agentful with Intelligent Detection (30 seconds)

```bash

```

**What happens during init:**

agentful analyzes your project and **automatically detects the best structure**:

**For Complex Projects:**

### How Detection Works

agentful automatically chooses the best structure based on:

| Criteria | Simple (Flat) | Complex (Hierarchical) |
|----------|---------------|------------------------|
| **Domains detected** | < 3 domains | ‚â• 3 domains |
| **Frameworks** | 0-1 frameworks | ‚â• 2 frameworks |
| **Monorepo** | No | Yes (npm workspaces, pnpm, turborepo) |
| **Project type** | Learning, MVP, single-purpose | Multi-domain, team, long-term |

**Don't worry about the choice** - agentful does the hard work for you! You can switch between structures anytime if needed.

### What Gets Created

**Always created:**
- `.claude/` - 7 specialized agents, 5 commands, skills
- `.agentful/` - Runtime state tracking (gitignored)
- `CLAUDE.md` - Project instructions for Claude

**For simple projects (also supported):**
- `PRODUCT.md` - Single-file product spec at root

**For complex projects (preferred):**
- `.claude/product/index.md` - Main product specification
- `.claude/product/domains/*/index.md` - Domain-specific specs

## Step 2: Define Your Product (2 minutes)

Edit the product spec that agentful created for you. The file location depends on your project structure.

---

### For Simple Projects (Flat Structure)

If agentful created `PRODUCT.md` in your project root (also supported for simple workflows), edit it:

```markdown
# Product Specification

## Overview
A simple todo list application.

## Tech Stack
- **Frontend**: Next.js 14, TypeScript, Tailwind CSS
- **Backend**: Next.js API Routes
- **Database**: None (in-memory for now)
- **Testing**: Vitest

## Features

### 1. Add Todo - CRITICAL
**Description**: Users can add new todo items

**Acceptance Criteria**:
- [ ] Input field for new todos
- [ ] Add button
- [ ] Todo appears in list

### 2. Toggle Todo - HIGH
**Description**: Users can mark todos as complete

**Acceptance Criteria**:
- [ ] Click todo to toggle completion
- [ ] Visual indicator for completed todos

### 3. Delete Todo - MEDIUM
**Description**: Users can delete todos

**Acceptance Criteria**:
- [ ] Delete button on each todo
- [ ] Confirmation prompt
```

**That's it!** Skip to Step 3.

---

### For Complex Projects (Hierarchical Structure - Preferred)

If agentful created `.claude/product/` (preferred for most projects), edit `.claude/product/index.md`:

```markdown
# Product Specification

## Overview
A todo list application with user accounts and sharing.

## Domains
- Authentication
- Tasks
- Sharing

## Tech Stack
- **Frontend**: Next.js 14, TypeScript, Tailwind CSS
- **Backend**: Next.js API Routes
- **Database**: PostgreSQL with Prisma
- **Testing**: Vitest + Playwright
```

Then edit the domain files agentful created, like `.claude/product/domains/authentication/index.md`:

```markdown
# Authentication Domain

## Overview
Handle user registration, login, and session management.

## Features

### 1. User Registration - CRITICAL
**Description**: Users can create accounts with email/password

**Acceptance Criteria**:
- [ ] Email validation (required, valid format)
- [ ] Password min 8 characters, 1 uppercase, 1 number
- [ ] Unique email check
- [ ] Success: redirect to dashboard
- [ ] Error: show inline error message

### 2. User Login - CRITICAL
**Description**: Users can log in with credentials

**Acceptance Criteria**:
- [ ] Email/password input
- [ ] JWT token generation
- [ ] Session persistence
- [ ] Error handling for invalid credentials
```

Create additional domain files as needed:
- `.claude/product/domains/tasks/index.md` - Task CRUD operations
- `.claude/product/domains/sharing/index.md` - Share lists with others

### Switching Between Structures

You can change your mind anytime:

**Simple ‚Üí Complex:**
```bash
mkdir -p .claude/product/domains
mv PRODUCT.md .claude/product/index.md
# Split content into domain files
```

**Complex ‚Üí Simple:**
```bash
cat .claude/product/index.md > PRODUCT.md
cat .claude/product/domains/*/*.md >> PRODUCT.md
rm -rf .claude/product/
```

agentful auto-detects the change!

Don't micromanage. agentful works best when you:

‚úÖ **Do:**

## Understanding agentful's Intelligence

agentful is smart - it adapts to your project automatically:

### Automatic Structure Detection

**During init:**
- Analyzes your project structure
- Counts detected domains (features, modules, areas)
- Checks for multiple frameworks
- Detects monorepo setup
- Chooses optimal structure (flat vs hierarchical)

**During development:**
- Auto-detects which structure you're using
- Reads from `.claude/product/index.md` (preferred) OR `PRODUCT.md` (also supported)
- No configuration needed - just pick what fits your project

### Tech Stack Detection

**Automatically detects:**
- Frontend framework (Next.js, React, Vue, etc.)
- Backend framework (Express, NestJS, etc.)
- Database/ORM (Prisma, Drizzle, etc.)
- Testing tools (Vitest, Jest, Playwright)
- Styling approach (Tailwind, CSS Modules, etc.)
- Build tools (Vite, Webpack, tsup)

**Generates specialized agents:**
- Stack-specific agents (e.g., `nextjs-agent`, `prisma-agent`)
- Knows your framework's patterns and conventions
- Follows best practices for your stack

### Quality Gate Adaptation

Quality gates automatically adapt to your stack:

- **TypeScript project?** ‚Üí Checks for type errors
- **JavaScript project?** ‚Üí Skips type checking
- **ESLint configured?** ‚Üí Runs ESLint
- **Biome configured?** ‚Üí Runs Biome instead
- **Playwright installed?** ‚Üí Runs E2E tests
- **No E2E framework?** ‚Üí Skips E2E checks

## Troubleshooting

### Issue: agentful seems stuck

**Solution:** Check status
```
/agentful-status
```

If iterations > 20 without progress:
1. Check for pending decisions
2. Run `/agentful-decide`
3. Or restart: `/agentful-start`

### Issue: Not making progress on features

**Solution:** Review your product spec (`.claude/product/index.md` or `PRODUCT.md`)
- Are acceptance criteria clear?
- Is tech stack specified?
- Try breaking features into smaller chunks

### Issue: Too many validation errors

**Solution:** Let Fixer work
- agentful will auto-fix most issues
- If stuck, run `/agentful-validate` to see specific errors
- Some errors may require manual intervention

2. **Autonomous Mode** (`/agentful-start`) - Hands-off development, 24/7 capability

**24/7 Development:**
```bash
/ralph-loop "/agentful-start" --max-iterations 50 --completion-promise "AGENTFUL_COMPLETE"
```

**Quick Decision Guide:**
| Need | Command |
|------|---------|
| Start building | `/agentful-start` |
| Check progress | `/agentful-status` |
| Answer questions | `/agentful-decide` |
| Make changes mid-flight | `/agentful` (conversation mode) |
| Run quality checks | `/agentful-validate` |
