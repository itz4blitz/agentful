# Product-Tracking Skill

The **product-tracking skill** monitors and calculates completion progress for your entire product, tracking features, subtasks, and quality gates.

## What It Does

- **Tracks feature completion** - Calculates percentage complete for each feature
- **Monitors quality gates** - Tracks test status, type checking, linting, coverage, security
- **Supports both structures** - Works with flat PRODUCT.md and hierarchical .claude/product/
- **Calculates overall progress** - Provides total project completion percentage
- **Reports by priority** - Shows CRITICAL, HIGH, MEDIUM, LOW feature status

## How It Works

### Data Structure

The skill tracks progress in `.agentful/completion.json`:

```json
{
  "features": {
    "User Authentication": {
      "status": "complete",
      "progress": 100,
      "subtasks": {
        "registration": "complete",
        "login": "complete",
        "password_reset": "complete"
      }
    },
    "User Profiles": {
      "status": "in_progress",
      "progress": 45,
      "subtasks": {
        "profile_page": "complete",
        "avatar_upload": "in_progress",
        "settings": "pending"
      }
    }
  },
  "gates": {
    "tests_passing": true,
    "no_type_errors": false,
    "no_dead_code": true,
    "coverage_80": false,
    "security_clean": true
  },
  "overall": 67
}
```

**Field Definitions:**
- `features` - Each feature with status and subtask breakdown
  - `status` - One of: `pending`, `in_progress`, `complete`, `blocked`
  - `progress` - Percentage complete (0-100)
  - `subtasks` - Individual subtask statuses
- `gates` - Quality gate pass/fail status (boolean)
- `overall` - Total project completion percentage (0-100)

### Progress Calculation

**Feature Progress:**
```
progress = (complete_subtasks / total_subtasks) √ó 100
```

**Overall Progress:**
```
overall = average(all_feature_progress) √ó quality_gate_factor
```

### Quality Gate Tracking

The skill monitors these gates:

| Gate | Check | Threshold |
|------|-------|----------|
| **Tests** | All tests passing | 100% must pass |
| **Type Checking** | No type errors | Adapts to stack |
| **Linting** | No lint errors | 0 errors |
| **Dead Code** | No unused code | 0 unused exports/imports |
| **Coverage** | Code coverage | ‚â•80% |
| **Security** | No vulnerabilities | 0 known issues |

## Priority Weighting

Features are prioritized by:

| Priority | Weight | Description |
|----------|--------|-------------|
| **CRITICAL** | 4x | Must have for launch |
| **HIGH** | 3x | Important for MVP |
| **MEDIUM** | 2x | Nice to have |
| **LOW** | 1x | Future consideration |

**Example:**
```
Overall Progress = weighted_average(feature_progress)

= (CRITICAL √ó 100 + HIGH √ó 80 + MEDIUM √ó 20 + LOW √ó 0) / (4 + 3 + 2 + 1)
= (400 + 240 + 40 + 0) / 10
= 68%
```

## Best Practices

### 1. Define Clear Subtasks
```markdown
### User Authentication - CRITICAL
- [ ] Registration form - CRITICAL
- [ ] Login page - CRITICAL
- [ ] Password reset - HIGH
- [ ] Remember me - MEDIUM
```

**Result:** Progress calculated accurately

### 2. Use Priorities Consistently
Mark each feature with a priority to help orchestrator select next task.

### 3. Update Status Regularly
The skill automatically updates when agents complete tasks.

## Troubleshooting

### Issue: "Progress stuck at 0%"
**Cause:** Features not properly defined in PRODUCT.md
**Solution:** Ensure features have subtasks with checkboxes

### Issue: "Overall progress doesn't match"
**Cause:** Quality gates failing
**Solution:** Run `/agentful-validate` to fix quality issues

### Issue: "Feature not showing in status"
**Cause:** Feature name mismatch between spec and completion tracking
**Solution:** Use exact feature names from PRODUCT.md
