# Development Workflow Pipeline

An interactive visualization showing the complete development pipeline from product specification to deployment.

## Overview

The Development Workflow Pipeline demonstrates how agentful manages the entire software development lifecycle through six distinct stages, with automated quality gates and decision points ensuring high-quality output at every step.

## Pipeline Stages

### 1. Product Specification üìã
The **Product Analyzer** agent evaluates your product specification for completeness, clarity, and feasibility. It identifies gaps, ambiguities, and calculates a readiness score before development begins.

**Key Activities:**
- Parse and analyze `.claude/product/index.md`
- Detect missing requirements or unclear acceptance criteria
- Calculate product readiness score (0-100%)
- Generate recommendations for improvement

### 2. Implementation ‚öôÔ∏è
The **Architect**, **Backend**, and **Frontend** agents collaborate to build your application based on the specification.

**Key Activities:**
- Architect detects tech stack and designs system architecture
- Backend implements APIs, database schemas, and business logic
- Frontend builds UI components, pages, and client-side interactions
- All agents follow established patterns and conventions

### 3. Testing üß™
The **Tester** agent writes comprehensive tests covering all code paths and user scenarios.

**Key Activities:**
- Write unit tests for individual functions and components
- Write integration tests for API endpoints and workflows
- Write E2E tests for critical user journeys
- Achieve minimum 80% code coverage

### 4. Quality Gates üîç
The **Reviewer** agent runs automated quality checks. If any check fails, the **Fixer** agent automatically attempts remediation.

**Six Core Quality Gates:**
1. **Type Checking** - No TypeScript/Flow errors
2. **Linting** - Code follows style guide (ESLint/Prettier)
3. **Tests** - All tests passing
4. **Coverage** - Minimum 80% code coverage
5. **Security** - No vulnerable dependencies
6. **Dead Code** - No unused exports or files

**Decision Point:** All checks pass?
- **Yes** ‚Üí Proceed to Review
- **No** ‚Üí Auto-fix loop (Fixer agent resolves issues)

### 5. Final Review ‚úÖ
The **Reviewer** agent performs final validation before deployment, including manual checks and documentation review.

**Key Activities:**
- Review code quality and architecture decisions
- Verify acceptance criteria met
- Check documentation completeness
- Validate production readiness

**Decision Point:** Approved?
- **Yes** ‚Üí Proceed to Deploy
- **No** ‚Üí Return to Implementation

### 6. Deploy üöÄ
The **Orchestrator** agent coordinates deployment to staging and production environments.

**Key Activities:**
- Build production optimized bundle
- Deploy to staging environment
- Run smoke tests
- Deploy to production
- Monitor for issues

## Key Features

### Automated Fixer Loop
When quality gates fail, the **Fixer** agent automatically attempts to resolve issues:
- Fix type errors by adding proper type annotations
- Resolve linting issues (formatting, unused variables)
- Fix failing tests or add missing test cases
- Update dependencies to resolve security vulnerabilities
- Remove dead code and unused exports

The fixer loop is visualized as a **red dashed edge** from the Quality Gates decision point back to Implementation.

### Human-in-the-Loop Checkpoints
While agentful automates most of the development pipeline, it includes strategic human checkpoints:
- **Product Specification:** Review and approve before development
- **Quality Gates:** Review failures that fixer cannot resolve
- **Final Review:** Approve before deployment
- **Deploy Decision:** Confirm production deployment

### Real-time Progress Tracking
Each stage shows:
- **Status badge:** Complete (‚úì), In Progress (‚ü≥), Pending (‚óã), Failed (‚úï)
- **Progress bar:** Visual percentage complete
- **Time spent:** Duration agent worked on this stage
- **Sub-steps:** Detailed breakdown of activities

Run `/agentful-status` to see your current pipeline progress in the CLI.
